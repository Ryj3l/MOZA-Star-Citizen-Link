# T-24: Phase 1 documentation pack

- **Task ID:** T-24
- **Title:** Produce user-facing and operator-facing documentation for the Phase 1 portable build
- **Depends on:** T-23 (final tuned build)
- **Phase:** 1
- **Estimated effort:** M (2–6 hours)
- **Branch:** `chore/24-phase1-docs`

## Context

Phase 1 ships as a private beta. The beta cohort needs: a quick-start guide, an FAQ, a known-issues list, and a troubleshooting page. Internal operators need a runbook for triaging support requests.

## Deliverables

1. `docs/user/QUICKSTART.md` — install and first-run, including:
   - System requirements (Windows 10 19041+, .NET 8 desktop runtime, MOZA AB6 or AB9)
   - Download the portable zip, extract, launch the .exe
   - First-launch flow (auto-detect Game.log, allow firewall prompt for log access)
   - How to fire the Test Sequence
   - How to use the emergency stop hotkey
   - Where settings and logs live
2. `docs/user/FAQ.md` — anticipated questions:
   - "Will this get me banned by EAC?" → Explain explicitly: this product never inspects or modifies the Star Citizen process. It only reads Game.log (which CIG itself produces and is intended for player diagnostic use). The product is not anti-cheat-adjacent. (Reference PRP §13.3.)
   - "My device doesn't show up." → Allowlist + driver troubleshooting.
   - "Effects feel too strong / too weak." → Master gain and per-category gain controls.
   - "Can I use a non-MOZA wheelbase?" → No, Phase 1. Subject to allowlist expansion.
   - "Does this work in multiplayer?" → Yes; it is purely an output-side companion app.
   - "I don't have a MOZA flight base; can I still try it?" → Preview mode.
3. `docs/user/KNOWN_ISSUES.md` — list from T-23's defect log (major and below).
4. `docs/user/TROUBLESHOOTING.md` — log paths, settings path, how to file a diagnostic bundle, how to reset settings.
5. `docs/ops/SUPPORT_RUNBOOK.md` — for whoever fields beta support requests:
   - How to ask a user to capture a diagnostic bundle
   - Common issues and resolutions
   - Escalation path
   - When to file a bug vs a feature request
6. `README.md` at repo root, updated to describe Phase 1 scope, build instructions, and link to user docs.
7. `CHANGELOG.md` at repo root, with the Phase 1 v0.1.0 entry summarizing all 24 tasks.

## Implementation notes

The EAC FAQ entry is the most consequential single piece of writing in this task. Get it reviewed carefully. The product's posture is:

> "MOZA Star Citizen Link runs as a separate Windows process. It reads the Game.log file that Star Citizen itself writes to disk. It does not attach to the Star Citizen process, does not read or write Star Citizen's memory, does not inject DLLs, and does not modify game files. All of its behavior is functionally equivalent to a third-party app that watches a log file your computer is already producing."

This wording is intentionally conservative. Do not promise that EAC will tolerate the app forever — anti-cheat policies can change. The promise is about what the app does, not what CIG / EAC will permit.
