## Implementation notes

Recommended sequence:

1. Move files in dependency order: leaf projects (Core, Models) first, then libraries (Logs, DirectInput, Profiles), then App last.
2. Use `git mv` to preserve file history.
3. After each move, run `dotnet build` to catch reference issues incrementally.
4. Update `namespace` declarations with find/replace, being careful with `using` directives in dependent files.
5. The `event-patterns.json` file is copied with the `CopyToOutputDirectory="PreserveNewest"` metadata still set in the new `.csproj`.
