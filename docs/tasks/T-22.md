# T-22: CI pipeline with forbidden-API scan

- **Task ID:** T-22
- **Title:** Stand up GitHub Actions CI with build, test+coverage, forbidden-API scan, DDR completeness check, pattern lint, preview build
- **Depends on:** T-03 (centralized packages), T-05 (DDRs)
- **Phase:** 1
- **Estimated effort:** S (1–2 hours, given the starters)
- **Branch:** `chore/22-ci-pipeline`

## Context

The starter bundle includes:
- `.github/workflows/ci.yml`
- `scripts/forbidden-api-scan.ps1`
- `scripts/check-ddr-completeness.ps1`
- `scripts/check-coverage.ps1`
- `scripts/validate-patterns.ps1`

This task installs them, adapts paths if the repo layout differs from the starter assumptions, and verifies they pass against the in-flight codebase.

## Deliverables

1. `.github/workflows/ci.yml` committed.
2. All four scripts under `scripts/` committed and `chmod +x`-friendly (PowerShell scripts run via `pwsh`, no chmod needed on Windows).
3. `scripts/package-portable.ps1` added — produces a portable zip via `dotnet publish` for PR preview artifacts:

   ```powershell
   param([string]$Configuration = "Release")
   $ErrorActionPreference = "Stop"
   $publishDir = "artifacts/publish"
   if (Test-Path $publishDir) { Remove-Item $publishDir -Recurse -Force }
   dotnet publish src/Moza.ScLink.App/Moza.ScLink.App.csproj `
       --configuration $Configuration `
       --output $publishDir `
       --runtime win-x64 `
       --self-contained false `
       /p:PublishSingleFile=false
   $zipName = "MozaStarCitizenLink-preview-$(Get-Date -Format yyyyMMdd-HHmmss).zip"
   Compress-Archive -Path "$publishDir/*" -DestinationPath "artifacts/$zipName" -Force
   ```
4. Branch protection rule on `main` requires all CI checks to pass before merge.
5. First green build on the branch.

## Acceptance criteria

- [ ] CI runs on every pull request and every push to `main`.
- [ ] Build job passes with `TreatWarningsAsErrors=true`.
- [ ] Tests pass.
- [ ] Coverage check passes (with thresholds matching starter; can be tuned down for early Phase 1).
- [ ] Forbidden-API scan passes (no SharpDX, no banned P/Invokes).
- [ ] DDR completeness check passes.
- [ ] Pattern lint runs (allowed to soft-fail in Phase 1 via `continue-on-error`).
- [ ] PR builds produce a downloadable portable preview zip.
- [ ] Branch protection enabled.

## Implementation notes

The pattern-lint step is marked `continue-on-error: true` for Phase 1 because the patterns themselves are placeholders. Flip this to `false` once T-11 ships and real patterns are in place with an `expectations.json` corpus.

For the preview zip, do not include PDBs in the zip distributed to users (file size concerns). Strip via `/p:DebugType=none /p:DebugSymbols=false` if needed, but keep symbols available as a separate CI artifact for later crash analysis.
