# T-20: Pattern hot reload

- **Task ID:** T-20
- **Title:** Pattern library hot reload with safe fallback
- **Depends on:** T-11
- **Phase:** 1
- **Estimated effort:** S (1–2 hours)
- **Branch:** `feat/20-pattern-hot-reload`

## Context

T-11 introduces a versioned pattern library. This task hardens hot reload so beta users (and developers) can iterate on patterns by editing JSON and seeing the change without restarting the app.

The trick: a malformed or regex-invalid edit must not crash the sensor. The current pattern set stays in effect until a valid new set arrives.

## Deliverables

1. `Moza.ScLink.Logs/Patterns/PatternFileWatcher.cs` — `FileSystemWatcher` wrapper with:
   - 300 ms debounce after the last change event
   - Read-the-file-with-retry (because text editors sometimes hold a write lock briefly)
   - Validation pipeline: parse JSON → check schema version → compile every regex with 100 ms timeout → if any step fails, log warning and keep current set
2. On successful reload, log at `Information`: "Pattern library reloaded: N patterns from <file>".
3. On failure, log at `Warning`: "Pattern reload failed: <reason>. Continuing with previous pattern set."
4. Reload events are surfaced to `DiagnosticsViewModel` so the UI can show "Patterns last reloaded: HH:MM:SS".
5. Unit tests:
   - `PatternFileWatcherTests.Debounces_Multiple_Changes`
   - `PatternFileWatcherTests.Retries_On_IOException`
   - `PatternFileWatcherTests.Falls_Back_On_Malformed_Json`
   - `PatternFileWatcherTests.Falls_Back_On_Invalid_Regex`
   - `PatternFileWatcherTests.Successful_Reload_Replaces_Set`

## Acceptance criteria

- [ ] Editing the file via Notepad triggers reload within 1 second of save.
- [ ] Saving a syntactically broken JSON does NOT crash the app and does NOT poison the in-memory pattern set.
- [ ] Saving a regex with catastrophic backtracking does NOT hang — the compile timeout (100 ms) catches it.
- [ ] Diagnostics UI reflects the last reload time.
- [ ] Tests pass.

## Implementation notes

`FileSystemWatcher` fires multiple events for a single save (Changed, Renamed for atomic-saves, etc.). Debounce by latching on the last event time and only acting after 300 ms of quiet.

Use `Regex.MatchTimeout` on every compiled `Regex` so runtime matching also has the safety:

```csharp
var compiled = new Regex(p.pattern,
    RegexOptions.IgnoreCase | RegexOptions.CultureInvariant | RegexOptions.Compiled,
    TimeSpan.FromMilliseconds(100));
```

A separate Phase 2 task can add a UI dialog to validate patterns interactively against a sample corpus.
