# T-14: Effect resolver with gain stack

- **Task ID:** T-14
- **Title:** Implement `IEffectResolver` that maps `GameEvent` → `ForceCommand[]`, applies the gain stack from PRP §5.9, and emits to the ForceCommand channel
- **Depends on:** T-06, T-13
- **Phase:** 1
- **Estimated effort:** M (2–6 hours)
- **Branch:** `feat/14-effect-resolver`

## Context

The existing `ForceFeedbackController` switches on `ScEventKind` and constructs `ForceEffect` instances inline. PRP §5.7 and §5.9 abstract this into an `IEffectResolver` with a gain stack that combines base effect intensity, event modifier, ship profile, category gain, master gain, and per-device multiplier.

## Deliverables

1. `Moza.ScLink.Effects/EffectResolver.cs` implementing `IEffectResolver`:
   - Looks up the effect for the `GameEventType` (mapping table; multiple effects per type allowed)
   - Computes final intensity via the gain stack
   - Applies envelope from the catalog
   - Returns `IReadOnlyList<ForceCommand>` (typically one `PlayEffectCommand`, sometimes a `StopEffectCommand` for state-transitioning events like `AtmosphereExited`)
2. `Moza.ScLink.Effects/GainStack.cs` as a pure static function implementing PRP §5.9:

   ```csharp
   public static double Compute(
       double baseEffectIntensity,
       double eventIntensityModifier,
       double shipProfileMultiplier,
       double categoryGain,
       double masterGain,
       double deviceGainMultiplier,
       double deviceMaxIntensityRecommended)
   ```

3. `Moza.ScLink.Effects/GameEventToEffectMap.cs` mapping `GameEventType` → effect ID(s) from the catalog. Phase 1 mappings:
   - `QuantumSpoolStarted` → `quantum-spool-v1`
   - `QuantumSpoolEnded` → stop `quantum-spool` state
   - `QuantumJumpExit` → `quantum-jump-exit-v1`
   - `AtmosphereEntered` → `atmosphere-entry-v1`
   - `AtmosphereExited` → stop `atmosphere` state
   - `LandingGearContact` → `landing-contact-v1`
   - `WeaponFireBallistic`, `WeaponFireEnergy`, `WeaponFireGeneric` → `weapon-fire-generic-v1`
   - `VehicleDestruction` → `vehicle-destruction-v1`
4. A hosted service `EffectResolverService` that reads from `GameEventReader` and writes to `ForceCommands`.
5. Unit tests:
   - `GainStackTests.cs`: 100% branch coverage on the formula, including clamping, rate-of-change, sustained cap
   - `EffectResolverTests.cs`: each Phase 1 event type produces the expected command
   - Stop events produce `StopEffectCommand` with correct `StateKey`
   - Unknown event type produces empty list (no command)

## Acceptance criteria

- [ ] Coverage on `Moza.ScLink.Effects` overall ≥ 90%.
- [ ] Unit tests pass.
