## Context

The R11 closeout call ruled #83 **fix-before-ship** (a silently non-functional safety control
is not shippable under documentation; PRP §15's permissive bar covers playback, not safety
controls) and scoped the family fix at input (d). Parent synthesis: **the e-stop halts the
device but resets no stage state — limiter, effect-cache, or device-side.** Three siblings
plus one load-bearing rider:

- **#83 (safety):** device-wide StopAll no-ops under contention
  (`DIERR_NOTEXCLUSIVEACQUIRED`) while the play path re-acquires — 22/22 no-ops 2026-06-08;
  empirical verdict **persists-until-Reset** (no designed or incidental halt at any
  acquisition boundary; ownerless persistence at owner death).
- **#84 (availability):** limiter `_active` never reset by e-stop; ~2–4 e-stop cycles wedge
  playback; restart-only recovery.
- **#88 (both axes):** post-e-stop cached replay `Start()` returns S_OK and renders nothing
  (silent failure); an unrelated fresh CreateEffect un-mutes it minutes later (delayed
  surprise force; resurrection ×1 observed).
- **#50 (rider, load-bearing):** sustained-effect expiry — REQUIRED for the §(g) post-fix
  full-G3-as-written validation; sustained plays accumulate to the cap regardless of e-stops,
  so a fixed build still wedges a 4-hour as-written soak without it. Excluding #50 would
  silently void a §(g) commitment — ruled IN, eyes-open.

## Design preamble (REQUIRED, per the T-12/T-15/T-16/T-17 fork-at-preamble pattern)

Mechanism forks resolve at the preamble with surface-then-confirm. Starting hypotheses
recorded here are NOT rulings:

1. **#88:** invalidate-and-dispose the effect cache on e-stop — routes every post-e-stop
   replay through the empirically-working fresh-create path and removes the resurrection
   substrate (stale device handles). Alternatives to weigh: re-download-on-replay; targeted
   invalidation.
2. **#84:** the e-stop must structurally reach the limiter it currently bypasses (the channel
   bypass goes device-only; `EffectResolverService` has zero e-stop handling;
   `SafetyLimiterStage.ApplyToState`'s default arm passes `StopAllCommand` through without
   mutating `_active`). Alternatives: e-stop subscription clearing `_active`;
   `StopAllCommand` mutating limiter state; both.
3. **#83:** stop path mirrors the play path's M9 re-acquire (on
   `DIERR_NOTEXCLUSIVEACQUIRED` → re-acquire → Reset/StopAll). Probe-validated mechanism:
   acquire + FFB Reset halted force in all observed conditions including ownerless.
4. **#50:** expiry shape for sustained entries (`AbsoluteMaxEffectDuration` per the original
   T-15 deferral; T-16 ownership note on the issue) + the G3-full sufficiency question (the
   #32 gate above).
5. **Cross-cutting:** replace-on-StateKey vs add-per-CommandId divergence (device REPLACES
   per StateKey; limiter ADDS per CommandId — validated live, D2) — decide whether T-30
   narrows it or records it as designed behavior.
6. **§(e) rider — load-path bound re-derivation:** re-derive the analytical e-stop load-path
   bound on the fixed build (new realistic depth under #50 expiry; new per-cycle stop cost
   under re-acquire+Reset) — one-paragraph analysis delivered with T-30's validation
   evidence. The pre-fix characterization (wake-immediate / execution-non-preemptive;
   theoretical 64 ms ceiling; realistic ≤4 / ~4 ms per the D2 correction) describes a path
   this fix changes.
7. **#95 finite-duration probe (EARLY — at design open, before committing the #95 mitigation):**
   Does the MOZA device honor finite DI effect durations? Probe EARLY at design open (DI
   acquire-probe path), before committing the #95 mitigation. Outcome decides whether #95
   self-arrests app-side (finite duration + keep-alive re-trigger, architecturally adjacent to
   #50 sustained-expiry) or acquires a #94 firmware dependency.

## Validation gate (carried from §(a)/§(g)/§(c) — the fix is NOT efficacious on bench alone)

- **Bench:** unit + integration green; e-stop/replay cycles clean — no wedge at 20 cycles, no
  silent replays, no resurrection.
- **One SC-contention in-game session:** e-stop while force plays, SC foreground — the
  unreproduced-displacement regime. Shares ONE rig sitting with: full G3-as-written (§(g))
  and A3/A4/A5 re-verification (#71 PR, §(c)).
- **§(e) rider:** the SC-contention session ALSO captures contended-path e-stop halt latency
  (activation timestamp → physical-halt evidence) — the session is already measuring
  efficacy; timing rides the same evidence.
- **#95 safety AC (admitted, gated post-fix):** the shared sitting ALSO validates — "a leaked
  or resident effect can never produce sustained uncommanded force surviving process death":
  effect left resident → abnormal process termination (hard kill) → base self-arrests within a
  bounded window, no manual power-cycle and no Cockpit/firmware intervention; paired clean
  handle-release control; finger on the power switch, bounded duration. Validated post-fix;
  rides this same SC-contention sitting (delivered by whichever car owns the #95 fix per the
  placement decision above).
- Env block before any hardware trigger (#86 discipline); the DI acquire-probe (`dc5ba79`)
