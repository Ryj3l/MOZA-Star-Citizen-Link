# T-05: Pre-approved-dependency DDRs

- **Task ID:** T-05
- **Title:** Verify and commit DDRs for the pre-approved Phase 1 dependencies
- **Depends on:** T-03
- **Phase:** 1
- **Estimated effort:** S (1–2 hours)
- **Branch:** `chore/05-pre-approved-ddrs`

## Context

PRP §4.0 requires every `<PackageReference>` to either be in the pre-approved Phase 1 list (PRP §4.2) or have a DDR. The pre-approved list documents which DDRs we owe ourselves. The starter bundle includes DDR drafts for these packages; this task is the formal commit and verification.

## Deliverables

1. Confirmed DDRs in `docs/decisions/` for each pre-approved package listed in PRP §4.2 (the starter bundle provides drafts):
   - `ddr-communitytoolkit-mvvm.md`
   - `ddr-serilog.md`
   - `ddr-serilog-sinks-file.md`
   - `ddr-serilog-settings-configuration.md`
   - `ddr-microsoft-extensions-hosting.md`
   - `ddr-microsoft-extensions-dependencyinjection.md`
   - `ddr-vortice-directinput.md`
   - `ddr-system-text-json.md`
   - `ddr-xunit.md`
   - `ddr-fluentassertions.md`
   - `ddr-nsubstitute.md`
   - `ddr-coverlet-collector.md`
2. ADR-0001 (Vortice.DirectInput) committed in `docs/decisions/ADR-0001-vortice-directinput.md` (provided in starter bundle).
3. `NOTICE.md` at repo root listing third-party attributions (MIT, Apache-2.0, BSD-3) per the licenses referenced in each DDR.
4. CI script `scripts/check-ddr-completeness.ps1` passes.

## Acceptance criteria

- [ ] Each DDR file exists with all template sections completed.
- [ ] CI DDR-completeness check passes.
- [ ] `NOTICE.md` contains the SPDX identifier and a short attribution line per package.
- [ ] PR description includes the file list and a confirmation that the FluentAssertions pin to 6.x is documented.

## Implementation notes

The starter bundle includes drafts. This task validates them against current package realities (latest versions, license confirmations, no recent CVEs) and commits them. Re-check each package's latest release date and SPDX license — package authors occasionally change licensing.

For `NOTICE.md`, this format is sufficient:

```markdown
# Third-party notices

This product uses the following third-party packages:

## CommunityToolkit.Mvvm (MIT)
Copyright (c) .NET Foundation and Contributors. https://github.com/CommunityToolkit/dotnet

## Serilog (Apache-2.0)
Copyright (c) Serilog Contributors. https://github.com/serilog/serilog

... (repeat per package)
```
