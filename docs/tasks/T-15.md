# T-15: Safety limiter with constants

- **Task ID:** T-15
- **Title:** Implement the safety limiter from PRP §5.8 between the resolver and the output worker
- **Depends on:** T-06, T-14
- **Phase:** 1
- **Estimated effort:** M (2–6 hours)
- **Branch:** `feat/15-safety-limiter`

## Context

PRP §5.8 specifies numeric safety constants that bound effect output: max intensity, max sustained intensity, rate-of-change limit, simultaneous-effect cap, startup ramp, stop ramp, emergency-stop max latency, absolute max duration. The safety limiter component inspects every `ForceCommand` before it reaches the output worker and enforces these constants.

## Deliverables

1. `Moza.ScLink.Effects/SafetyLimiter.cs` exposing:

   ```csharp
   public interface ISafetyLimiter
   {
       ForceCommand Apply(ForceCommand command, SafetyContext context);
   }

   public sealed record SafetyContext(
       IReadOnlyList<PlayEffectCommand> ActiveEffects,
       DeviceCapabilities Device,
       DateTimeOffset Now,
       IReadOnlyDictionary<string, double> RecentIntensityHistory);
   ```

2. The implementation enforces:
   - Clamp intensity to `[0, deviceCapabilities.MaxIntensityRecommended]`
   - For sustained effects (IsSustained=true, Duration > 5s OR Duration == Zero): clamp intensity to `MaxSustainedIntensity` (0.7)
   - Rate-of-change limit: intensity cannot change by more than 4.0 / second (measured against the last command for the same effect ID)
   - Simultaneous-effect cap: if `ActiveEffects.Count >= MaxSimultaneousEffects` (4), preempt the oldest non-sustained effect
   - Effects with `Duration > AbsoluteMaxEffectDuration` (10 minutes) have their duration capped
3. Every limit application is logged at `Warning` level with a structured event including the original and clamped values.
4. The limiter is wired between `EffectResolver` and `ForceCommands` channel (i.e., resolver → limiter → channel).
5. Unit tests — each constant in §5.8 has a dedicated test that asserts the limiter enforces it:
   - `SafetyLimiterTests.Clamps_Intensity_Above_DeviceMax`
   - `SafetyLimiterTests.Caps_Sustained_Intensity_At_Point_Seven`
   - `SafetyLimiterTests.Enforces_RateOfChange_Limit`
   - `SafetyLimiterTests.Preempts_Oldest_NonSustained_When_AtMaxSimultaneous`
   - `SafetyLimiterTests.Caps_Duration_At_AbsoluteMax`
   - `SafetyLimiterTests.Passes_Through_Commands_Below_Limits`
   - `SafetyLimiterTests.Logs_Warning_On_Each_Clamp`
