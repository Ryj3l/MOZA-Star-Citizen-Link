# T-21: Park existing screen-capture code in legacy/

- **Task ID:** T-21
- **Title:** Move `src/MozaStarCitizen.App/ScreenCapture/*` into `legacy/screen-capture/` outside the build; document the parking decision
- **Depends on:** T-02
- **Phase:** 1
- **Estimated effort:** S (under 1 hour)
- **Branch:** `chore/21-park-screen-capture`

## Context

The existing screen-capture code (`WindowsGraphicsCapture.cs`, related WGC interop) targets future use but is not part of Phase 1. PRP §3 makes `Moza.ScLink.Screen` an empty Phase 2 project. Rather than delete the existing code (it represents real WGC interop knowledge), park it in `legacy/` so it's available for reference and lift-and-shift in Phase 2.

## Deliverables

1. Files moved (via `git mv` to preserve history):
   - `src/MozaStarCitizen.App/ScreenCapture/*.cs` → `legacy/screen-capture/`
2. `legacy/screen-capture/README.md` documenting:
   - What this code does (WGC frame capture, periodic frame access)
   - Why it's parked (Phase 1 doesn't use screen capture; Phase 2 will revive)
   - The expected revival path: `Moza.ScLink.Screen/ScreenSensor.cs` implementing `ISensor`, calling into refactored capture code
   - Known issues / state of the code at parking time
3. The `legacy/` directory is excluded from the `.sln` and not built. The CI does NOT scan it for forbidden APIs (it's effectively documentation).
4. `.gitignore` does not exclude `legacy/`.
5. PR description references this task and explains the move.

## Acceptance criteria

- [ ] No references to `MozaStarCitizen.App.ScreenCapture.*` remain in built code.
- [ ] `legacy/screen-capture/` exists and contains the moved files plus a README.
- [ ] CI passes (legacy folder is ignored by forbidden-API scan).
- [ ] `dotnet build` does not attempt to compile legacy files.

## Implementation notes

To exclude `legacy/` from the forbidden-API scanner, ensure `scripts/forbidden-api-scan.ps1` has `\\legacy\\` in its excluded folders list (already present in the starter).

If your editor accidentally adds `legacy/screen-capture/*.cs` to the solution, remove those entries from `.sln`. A cleaner approach is to give legacy files an extension other than `.cs` (e.g., `.cs.parked`) but that loses syntax highlighting; the project preference is to keep `.cs` and rely on solution-file exclusion.
