# T-19: Settings persistence with schema versioning

- **Task ID:** T-19
- **Title:** Replace `AppSettingsStore` with a schema-versioned settings system; add diagnostic bundle export with redaction
- **Depends on:** T-04, T-18
- **Phase:** 1
- **Estimated effort:** M (2–6 hours)
- **Branch:** `feat/19-settings-and-diagnostic-bundle`

## Context

The existing `AppSettingsStore` reads/writes a flat `AppSettings` JSON with fault tolerance. PRP §11 requires schema versioning, migrations, atomic writes, and a separate "diagnostic bundle" export that redacts personally identifiable information.

## Deliverables

1. `Moza.ScLink.Profiles/Settings/AppSettings.cs` — settings record with explicit `int SchemaVersion { get; init; } = 1;`.
2. `Moza.ScLink.Profiles/Settings/SettingsStore.cs` — loads, migrates, saves. Persistence path: `%LOCALAPPDATA%\MozaStarCitizen\settings.json`. Atomic write via temp-file + `File.Move` with `File.Move(tempPath, finalPath, overwrite: true)`.
3. `Moza.ScLink.Profiles/Settings/SettingsMigrator.cs` — applies migrations from older versions to current. No migrations yet (this is v1) but the framework is present so v2 onward can drop in cleanly.
4. `Moza.ScLink.Diagnostics/Bundles/DiagnosticBundleExporter.cs` — produces a `.zip` containing:
   - Most recent 7 days of logs (already on disk)
   - Current settings.json (with sensitive fields scrubbed — see redactor)
   - Current effect catalog and rule library
   - System info (OS version, .NET version, device list)
   - A `MANIFEST.md` describing what is included
5. `Moza.ScLink.Diagnostics/Bundles/Redactor.cs` — scrubs:
   - Windows user profile paths (`C:\Users\<name>\` → `C:\Users\<REDACTED>\`)
   - Machine name in log lines
   - Any GUID that appears in known sensitive contexts (e.g., DirectInput device GUIDs are kept; user account GUIDs are scrubbed — Phase 1 has none of the latter)
   - Email addresses if any appear in logs
   - The full Game.log path is replaced with `<GAME_LOG_PATH>`
6. Settings UI: master gain slider (clamped 0..1 with 0.6 default), category gains, hotkey configuration, logging mode selector, force-preview-mode checkbox, emergency-stop reset.
7. Unit tests:
   - `SettingsStoreTests.RoundTrip_Default_Settings`
   - `SettingsStoreTests.Atomic_Write_Survives_Crash_Mid_Write` (simulate via abort on first write attempt)
   - `SettingsStoreTests.Schema_Mismatch_Triggers_Migration`
   - `SettingsStoreTests.Corrupted_Json_Falls_Back_To_Defaults_With_Warning`
   - `SettingsStoreTests.Master_Gain_Clamped_To_Valid_Range_On_Load`
   - `RedactorTests.Scrubs_User_Profile_Paths`
   - `RedactorTests.Scrubs_Machine_Names`
   - `RedactorTests.Preserves_Device_Guids`
   - `DiagnosticBundleExporterTests.Bundle_Contains_All_Required_Sections`
   - `DiagnosticBundleExporterTests.Bundle_Has_Manifest`

## Implementation notes

`File.Move(string, string, bool)` is atomic on NTFS. Combined with `File.Replace` for the case where a file already exists, the pattern is:

```csharp
public async Task SaveAsync(AppSettings settings, CancellationToken ct)
{
    var json = JsonSerializer.Serialize(settings, _jsonOptions);
    var tempPath = _settingsPath + ".tmp";
    var backupPath = _settingsPath + ".bak";

    await File.WriteAllTextAsync(tempPath, json, ct).ConfigureAwait(false);

    if (File.Exists(_settingsPath))
    {
        File.Replace(tempPath, _settingsPath, backupPath, ignoreMetadataErrors: true);
    }
    else
    {
        File.Move(tempPath, _settingsPath);
    }
}
```

For redaction, compose regexes carefully and unit-test corner cases (paths with spaces, UNC paths, paths in JSON-escaped form within log lines).

The bundle's MANIFEST.md should document: schema versions of each artifact, the SHA of the build that produced the bundle, the redactor version, the date range covered by logs, and a list of what is NOT included (audio captures, screenshots, full game log).
