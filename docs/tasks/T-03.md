# T-03: Centralized package versions

- **Task ID:** T-03
- **Title:** Adopt central package-version management via `Directory.Packages.props`
- **Depends on:** T-02
- **Phase:** 1
- **Estimated effort:** S (1–2 hours)
- **Branch:** `chore/03-centralized-package-versions`

## Context

The starter `Directory.Packages.props` from T-01 declares centralized package versions and sets `<ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>`. Individual `.csproj` files must reference packages by name only, never by version. This task verifies the rule across all 24 projects.

## Deliverables

1. `<ManagePackageVersionsCentrally>true</ManagePackageVersionsCentrally>` set in `Directory.Build.props` (already done in T-01 starter).
2. Every `<PackageReference>` in every `.csproj` references a package by `Include` only (no `Version` attribute on the `<PackageReference>` element).
3. Any package version not in the pre-approved list in PRP §4.2 has a DDR filed (see `docs/decisions/ddr-TEMPLATE.md`).
4. The forbidden-API analyzer (`Microsoft.CodeAnalysis.BannedApiAnalyzers`) is added to all production projects via `Directory.Build.props`.
5. The .NET analyzers (`Microsoft.CodeAnalysis.NetAnalyzers`) are inherited via `<AnalysisLevel>latest-Recommended</AnalysisLevel>` (already set in `Directory.Build.props`).

## Acceptance criteria

- [ ] `dotnet restore` succeeds with no NU1604 (PackageReference must have version) warnings/errors.
- [ ] `dotnet build --configuration Release` succeeds.
- [ ] CI script `scripts/check-ddr-completeness.ps1` passes.
- [ ] Forbidden-API analyzer runs (verified by introducing a temporary `using SharpDX;` in a throwaway file — build should fail — then revert).
- [ ] PR description lists any new DDRs filed.

## Implementation notes

To convert existing `<PackageReference>` entries:

```xml
<!-- Before -->
<PackageReference Include="Serilog" Version="4.0.0" />

<!-- After (version moves to Directory.Packages.props) -->
<PackageReference Include="Serilog" />
```

If MSBuild reports unused central package versions, that is acceptable: a package can be declared in `Directory.Packages.props` for future use without being referenced yet.
