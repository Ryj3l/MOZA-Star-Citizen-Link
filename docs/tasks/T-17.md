# T-17: Preview mode formalization

- **Task ID:** T-17
- **Title:** Formalize preview mode: app runs end-to-end without hardware, with a software preview that logs intended device output
- **Depends on:** T-07, T-14
- **Phase:** 1
- **Estimated effort:** S (1–2 hours)
- **Branch:** `feat/17-preview-mode`

## Context

The existing `FallbackForceFeedbackDevice` logs what would have been sent. PRP §13.3 wants this formalized as "preview mode" with explicit UI indication and a richer textual representation so developers (and beta users without hardware) can iterate on the catalog without a MOZA base attached.

## Deliverables

1. `Moza.ScLink.DirectInput/PreviewForceFeedbackDevice.cs` — implements `IForceFeedbackDevice`, logs every command with:
   - Wall-clock time
   - Effect ID
   - Final intensity
   - Frequency, duration, direction
   - Active-effects count after this command
2. The previewer also publishes to an in-app `IObservable<PreviewedCommand>` so the diagnostics panel shows recent activity.
3. Preview mode auto-engages when:
   - No allowlisted device is detected at startup, OR
   - The user explicitly toggles "Preview mode" in settings
4. UI banner: when in preview mode, a clearly visible banner reads "Preview mode — no device connected (or preview forced)". Color: amber.
5. Settings: `forcePreviewMode: bool` in `appsettings.json` and exposed in the settings UI.
6. Unit tests:
   - `PreviewForceFeedbackDeviceTests.Logs_Each_Command`
   - `PreviewForceFeedbackDeviceTests.Reports_Active_Count_Correctly`
   - `PreviewForceFeedbackDeviceTests.StopAll_Clears_Active_Effects`
   - `PreviewForceFeedbackDeviceTests.Stops_Effect_By_Id`

## Acceptance criteria

- [ ] Without hardware, app launches, monitors log, fires events, and shows a stream of "preview" entries.
- [ ] With hardware available but preview mode forced, no device commands actually reach the device.
- [ ] The diagnostic panel's preview view shows the last 50 commands in real time.
- [ ] Banner present in preview mode; absent when a real device is driven.
- [ ] Tests pass; coverage ≥ 80% on previewer.
