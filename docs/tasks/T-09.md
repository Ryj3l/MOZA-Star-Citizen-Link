# T-09: MVVM modernization with CommunityToolkit

- **Task ID:** T-09

- **Depends on:** T-04

- **Branch:** `feat/09-mvvm-modernization`

## Context

The existing `MainViewModel` hand-rolls `INotifyPropertyChanged` via a `SetField` helper. The existing `RelayCommand` is a manual `ICommand` implementation. CommunityToolkit.Mvvm provides source-generated equivalents that are smaller, faster, and less error-prone.

## Deliverables

1. `MainViewModel` refactored to inherit from `ObservableObject`.
2. Properties use `[ObservableProperty]` attribute on backing fields:

   ```csharp
   [ObservableProperty]
   private string _status = "Ready.";
   ```

3. Commands use `[RelayCommand]` attribute on methods:

   ```csharp
   [RelayCommand(CanExecute = nameof(CanStart))]
   private async Task StartAsync() { ... }

   private bool CanStart() => !IsMonitoring && !_isStarting && File.Exists(GameLogPath);
   ```

4. The hand-rolled `RelayCommand.cs` is deleted.
5. The hand-rolled `SetField` pattern is removed.
6. Existing UI bindings continue to work without XAML changes (verify by running the app and exercising every button).
7. Unit tests: `MainViewModelTests.cs` covering at least one property-change notification and one command can-execute change.

## Acceptance criteria

- [ ] App launches and behaves identically to before.
- [ ] All buttons enabled/disabled per their `CanExecute` predicates.
- [ ] Property changes propagate to the UI (verify Status, Events list, Diagnostics list).
- [ ] No references to the old `RelayCommand` class remain.
- [ ] `INotifyPropertyChanged` is not manually implemented anywhere in view models.
- [ ] Coverage on view models ≥ 40%.

## Implementation notes

When refactoring `RelayCommand`-based properties, CommunityToolkit's source generator names the public property by capitalizing the backing field's name (dropping the leading underscore). For example, `_status` → `Status`. Ensure XAML bindings still resolve.

For async commands, use `[RelayCommand]` on a method returning `Task`. CommunityToolkit wires this to `IAsyncRelayCommand` automatically. The `IsExecuting` property is exposed for free, which simplifies the existing `_isExecuting` bookkeeping.
