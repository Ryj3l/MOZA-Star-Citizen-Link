# T-04: Serilog bootstrap and AppLog shim

- **Task ID:** T-04
- **Title:** Introduce Serilog as the canonical logging infrastructure; keep `AppLog` as a thin shim during transition
- **Depends on:** T-03
- **Phase:** 1
- **Estimated effort:** M (2–6 hours)
- **Branch:** `chore/04-serilog-bootstrap`

## Context

PRP §2.8 mandates Serilog with local rolling files. The existing `AppLog` static class is a thin `File.AppendAllText` wrapper that does not support log levels, structured properties, or multiple sinks. This task replaces the underlying implementation while keeping the `AppLog.Write(...)` call sites compiling, so the migration is incremental.

## Deliverables

1. Serilog configured in `Moza.ScLink.App/Program.cs` (or equivalent host bootstrap):
   - Console sink in Debug builds
   - `Serilog.Sinks.File` with daily rolling, 14-day retention, 50 MB cap, output to `%LOCALAPPDATA%\MozaStarCitizen\logs\app-YYYYMMDD.log`
   - Default level: `Information`
   - Enrichers: `Process`, `Thread`, `MachineName` (machine name is for local logs only; redactor in T-XX scrubs it from exports)
2. Configuration via `appsettings.json` using `Serilog.Settings.Configuration`, with override files per logging mode (`appsettings.diagnostic.json`, `appsettings.developer-lab.json`).
3. `AppLog` shim in `Moza.ScLink.Core/Diagnostics/AppLog.cs`:

   ```csharp
   public static class AppLog
   {
       public static ILogger Logger { get; internal set; } = Log.Logger;

       public static void Write(string message) => Logger.Information("{Message}", message);

       public static void Write(Exception ex, string context) =>
           Logger.Error(ex, "{Context}", context);

       public static string LogPath => /* current Serilog file sink path */;
   }
   ```

4. All existing `AppLog.Write(...)` calls compile without change.
5. `Moza.ScLink.App` switched to `IHostBuilder` / `Microsoft.Extensions.Hosting` so Serilog integrates via `Serilog.Extensions.Hosting`.
6. Single-instance mutex logic from the old `App.xaml.cs` is preserved.
7. Unit tests for the file-sink path: clock-controllable test that asserts daily roll, size-cap roll, and 14-day retention deletion (the retention worker is a stub here; full implementation in a later task).

## Acceptance criteria

- [ ] App starts and logs to `%LOCALAPPDATA%\MozaStarCitizen\logs\app-YYYYMMDD.log` immediately after launch.
- [ ] Log lines include process ID, thread ID, level, timestamp, and message template.
- [ ] All existing `AppLog.Write(...)` calls compile and produce output at `Information` level (errors at `Error` level when an exception is passed).
- [ ] Single-instance mutex behavior preserved.
- [ ] Switching to diagnostic mode (via `appsettings.diagnostic.json` overlay) raises log level to `Debug`.
- [ ] Unit tests in `Moza.ScLink.Core.Tests/Diagnostics/SerilogConfigurationTests.cs` pass.
- [ ] Build clean at warning level 5.

## Implementation notes

Pattern for `Program.cs`:

```csharp
public static class Program
{
    [STAThread]
    public static int Main(string[] args)
    {
        Log.Logger = new LoggerConfiguration()
            .ReadFrom.Configuration(BuildConfiguration())
            .Enrich.WithProcessId()
            .Enrich.WithThreadId()
            .CreateBootstrapLogger();

        try
        {
            using var host = CreateHostBuilder(args).Build();
            AppLog.Logger = host.Services.GetRequiredService<ILoggerFactory>()
                                          .CreateLogger("App");

            var app = new App();
            app.InitializeComponent();
            return app.Run();
        }
        catch (Exception ex)
        {
            Log.Fatal(ex, "Host terminated unexpectedly");
            return 1;
        }
        finally
        {
            Log.CloseAndFlush();
        }
    }

    private static IHostBuilder CreateHostBuilder(string[] args) =>
        Host.CreateDefaultBuilder(args)
            .UseSerilog((ctx, services, cfg) => cfg
                .ReadFrom.Configuration(ctx.Configuration)
                .ReadFrom.Services(services));
}
```

`appsettings.json` shape:

```json
{
  "Serilog": {
    "Using": ["Serilog.Sinks.File"],
    "MinimumLevel": { "Default": "Information" },
    "WriteTo": [
      {
        "Name": "File",
        "Args": {
          "path": "%LOCALAPPDATA%\\MozaStarCitizen\\logs\\app-.log",
          "rollingInterval": "Day",
          "retainedFileCountLimit": 14,
          "fileSizeLimitBytes": 52428800,
          "rollOnFileSizeLimit": true,
          "outputTemplate": "[{Timestamp:yyyy-MM-ddTHH:mm:ss.fff zzz} {Level:u3}] [pid:{ProcessId} tid:{ThreadId}] {SourceContext}: {Message:lj}{NewLine}{Exception}"
        }
      }
    ],
    "Enrich": ["WithProcessId", "WithThreadId", "WithMachineName"]
  }
}
```

Environment-variable expansion in the path uses the standard `%LOCALAPPDATA%` Win32 pattern; verify on a fresh Windows account.
