## Empirically-verified substrate

**Vortice.DirectInput.Envelope** (reflection probe against installed `Release/net8.0-windows10.0.19041.0/Vortice.DirectInput.dll`):
- Unsealed class (not struct), base `System.Object`
- Private `Size : Int32` field (interop-managed; do not touch)
- Four public mutable `Int32` fields: `AttackLevel`, `AttackTime`, `FadeLevel`, `FadeTime`
- Single parameterless `.ctor()` ŌĆö object-initializer is the only construction pattern

**Catalog arithmetic invariant** (verified across `src/Moza.ScLink.Effects/Catalogs/phase1.json`):
- For fixed-duration effects (`durationMs > 0`): `durationMs == attackMs + holdMs + decayMs + releaseMs` exactly across all 5 fixed-duration effects
- For sustained effects (`durationMs == 0`): envelope phases describe attack-on-start + release-on-stop intent

This invariant is a substantive precondition for Choice-A's Duration-preservation property.

## Algorithm: Choice A ŌĆö Peak-anchored fold

```
Given:
  ForceEffect effect (carrying ForceEnvelope env)
  double finalIntensity in [0.0, 1.0]

Compute:
  envelopeIntensity = finalIntensity * env.AttackLevel       (peak-anchored)
  magnitude         = ScaleMagnitude(envelopeIntensity)      (existing helper; [0, 10000])

  Envelope:
    AttackLevel = 0                                          (start at zero)
    AttackTime  = ClampPositiveMicroseconds(env.Attack)      (env.Attack TimeSpan ŌåÆ ┬Ąs)
    FadeLevel   = 0                                          (end at zero)
    FadeTime    = ClampPositiveMicroseconds(env.Release)     (env.Release TimeSpan ŌåÆ ┬Ąs)

  Duration:
    DurationToMicroseconds(effect.Duration)                  (catalog Duration unchanged)
```

Hold + Decay fold into the implicit "body at main" phase of DirectInput's envelope semantics; SustainLevel is metadata-only at the device layer (catalog data preserved but not rendered).

**Total-function property (design requirement):** Choice A is a total function over all valid catalog inputs. For every `ForceEffect` with a non-null `ForceEnvelope` and a finite `finalIntensity`, `BuildEffectParameters` returns a valid `EffectParameters` instance. The `throw new NotSupportedException` block at lines 818-824 is **REMOVED entirely**, not converted to a different exception class. Verified empirically by unit tests over the full Phase-1 catalog matrix.
